# GUI改造計画（dev / user 2ビルド）

## 目的と制約
- 〇✖ロジックは開発チーム以外に漏らさない。
- 一般ユーザー版は逆コンパイルしてもロジックが分からない構成にする。
- GUIは **dev版（日本語）** と **user版（英語）** の2種類を用意する。

## ビルド方針
- **dev / user を別ビルド**で作成（配布ミス防止）。
- `GUI_VARIANT=dev|user` をビルド時に注入し、renderer側で表示を切替。

## dev版GUI（日本語）
### アクションバー
- 日本語化。
- 現在実行中の作業を頻繁に表示。

### メイン画面
- 残高 / セッション損益 / BET数 / 勝率 を明示。
- 「Developer View」→ **「ラプラスパネル」**。
- ラプラスパネル内の英語はすべて日本語化。
- SET HISTORY を見やすく再配置・視認性改善。
- DAILY PNL / LIVE FEED / RECENT HAND / LOG はすべて日本語化。

### ログ
- 開発チームが必要とする内容を中心に表示。
- 不要なデバッグログは削除。

### 設定
- ライセンスキー / テーブルフィルタータブは削除。
- 設定全体を日本語化。

## user版GUI（英語）
- 〇✖ロジック関連UIを完全削除。
- ラプラスパネルは削除。
- 日本語は一切使用しない（全て英語）。

## デザイン（共通）
- 全体を近未来的デザインに統一。
- ボタン/カード/ログ/パネルを統一スタイルに刷新。
- アクセント色・グラデーションを統一。

## 実装ステップ
1. 既存GUIの文言・パネル・設定項目を洗い出し。
2. dev/user 分岐（ビルド時定数）を導入。
3. dev版UIを日本語化＋ラプラスパネル改善。
4. user版UIからラプラスパネル・〇✖関連を削除。
5. 近未来デザインCSSの共通化。
6. user版ビルドでロジック露出が無いことを確認。
